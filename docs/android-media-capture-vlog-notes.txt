안드로이드 미디어 촬영 및 브이로그 클립 메모
============================================

현재 상태
---------
- 미션 인증은 이미지 제출만 활성화되어 있습니다.
- CameraX 사진 촬영과 Android Photo Picker 이미지 선택은 같은 이미지 제출 흐름을 사용합니다.
- 브이로그용 동영상 촬영 코드는 이미 구현되어 있고, 컴파일도 됩니다.
- 다만 현재는 기능 플래그로 일부러 꺼두었습니다.
- 동영상 업로드와 VIDEO 미션 파일 등록 코드도 들어가 있지만, 백엔드가 VIDEO 파일 등록을 확실히 받을 때까지 기본 비활성 상태입니다.

로컬 브이로그 클립 녹화 켜는 방법
---------------------------------
1. 아래 파일을 엽니다.
   frontend/app/src/main/kotlin/com/ssafy/culture/ui/screen/camera/QuestCameraScreen.kt

2. 아래 값을 찾습니다.
   private const val VlogVideoCaptureEnabled = false

3. 아래처럼 바꿉니다.
   private const val VlogVideoCaptureEnabled = true

켜면 동작하는 내용
------------------
- CameraX가 Preview + ImageCapture + VideoCapture를 함께 바인딩합니다.
- 사용자가 "촬영하기"를 누르면 카운트다운과 동시에 무음 mp4 녹화가 시작됩니다.
- 카운트다운이 끝나면 동영상 녹화를 멈추고 인증용 사진을 촬영합니다.
- 촬영 결과인 MediaCaptureResult에는 아래 값이 들어갑니다.
  - imageUri
  - videoUri
  - videoDurationMillis
- mp4 파일은 앱 cache에 아래 형태의 이름으로 저장됩니다.
  mission_{storyId}_{chapterId}_{timestamp}_clip.mp4

목업 VIDEO 미션 제출 켜는 방법
------------------------------
1. 아래 파일을 엽니다.
   frontend/app/src/main/kotlin/com/ssafy/culture/ui/screen/camera/QuestCameraScreen.kt

2. 아래 값을 찾습니다.
   private const val VlogVideoSubmitEnabled = false

3. 아래처럼 바꿉니다.
   private const val VlogVideoSubmitEnabled = true

켜면 동작하는 내용
------------------
- MediaCaptureResult.videoUri가 있으면 이미지 제출 뒤에 동영상도 업로드/등록합니다.
- 목업 모드에서는 MediaUploadRepository가 아래 형태의 가짜 URL을 반환합니다.
  mock://mission-videos/{storyId}/{chapterId}/{fileName}
- 이후 StoryRepository.submitMissionFile이 아래 타입으로 호출됩니다.
  type = "VIDEO"
- 이미지는 항상 먼저 제출되므로 기존 사진 인증 흐름은 유지됩니다.

주요 구현 위치
--------------
- frontend/app/src/main/kotlin/com/ssafy/culture/ui/screen/camera/QuestCameraScreen.kt
  - VlogVideoCaptureEnabled
  - VlogVideoSubmitEnabled
  - VideoCapture<Recorder> 생성
  - activeVlogRecording 상태
  - startVlogVideoRecording(...)
  - MediaCaptureResult(imageUri, videoUri, videoDurationMillis)

- frontend/app/src/main/kotlin/com/ssafy/culture/data/media/MediaModels.kt
  - MediaCaptureResult.videoUri
  - MediaCaptureResult.videoDurationMillis
  - UploadedMissionMedia.videoUri
  - UploadedMissionMedia.videoDurationMillis
  - MissionMediaType.Video

- frontend/app/src/main/kotlin/com/ssafy/culture/data/media/MediaUploadRepository.kt
  - createCameraVideoFile(...)
  - uploadMissionVideo(...)
  - 목업 VIDEO URL 생성
  - 실제 presigned URL 기반 video/mp4 업로드 경로

- frontend/app/src/main/AndroidManifest.xml
  - RECORD_AUDIO 권한은 일부러 주석 처리해두었습니다.
  - 현재 준비된 동영상 클립은 무음 녹화라 마이크 권한이 필요 없습니다.

오디오가 필요한 경우
--------------------
현재 준비된 동영상 경로는 무음 클립을 녹화합니다. 나중에 소리까지 담고 싶다면 아래 순서로 켜면 됩니다.

1. AndroidManifest.xml에서 RECORD_AUDIO 주석을 해제합니다.
2. 권한 온보딩 요청 목록에 Manifest.permission.RECORD_AUDIO를 추가합니다.
3. QuestCameraScreen.kt의 startVlogVideoRecording(...)에서 start(...) 전에 withAudioEnabled()를 붙입니다.
4. Android 12 이상에서 마이크 권한 거부 케이스를 반드시 테스트합니다.

백엔드 관련 메모
----------------
- 이번 작업에서는 백엔드를 수정하지 않았습니다.
- 백엔드 담당 범위 밖이므로 VIDEO 제출은 기본값으로 꺼두었습니다.
- 목업 모드에서는 VlogVideoSubmitEnabled만 true로 바꾸면 VIDEO 등록 흐름을 로컬에서 테스트할 수 있습니다.
- 실서버 모드에서 쓰려면 백엔드가 아래 API에서 type = "VIDEO"를 받아야 합니다.
  /stories/{storyId}/chapters/{chapterId}/files

VlogVideoCaptureEnabled를 켠 뒤 권장 테스트 순서
-----------------------------------------------
1. 빌드합니다.
   cd frontend
   ./gradlew.bat :app:assembleDebug

2. debug APK를 설치하고 앱을 실행합니다.
3. 퀘스트 상세 화면을 엽니다.
4. "카메라로 인증하기"를 누릅니다.
5. 카메라 권한을 허용합니다.
6. "촬영하기"를 누릅니다.
7. 카운트다운 뒤 사진 미리보기가 뜨는지 확인합니다.
8. 목업 모드에서 제출 후 결과 화면으로 이동하는지 확인합니다.
